# 06. システム全体の処理フロー

最終更新: 2026-04-25

このドキュメントでは、AIトレードボットがどのような順番で動き、どのような判断を行っているかを、システムに詳しくない方でもわかりやすいように説明します。

## 処理の全体の流れ

ボットはスケジュール（例：1時間に1回）に沿って、以下の「5つのステップ」を順番に自動で実行します。

### ステップ1：ニュースの収集（情報収集）
システムがインターネット上のニュースサイトやRSSフィードから、経済や株式に関する最新のニュース記事を自動で集めてきます。

### ステップ2：パニック度の測定（1次チェック）
集めたニュースを一つずつ、AI（Gemini）に読ませて、「この記事は市場にどれくらいパニック（不安や過熱感）を引き起こしそうか？」を0〜100の点数（パニックスコア）で評価させます。
- **点数が低い（70点未満）**：影響が少ないニュースとして、これ以上の詳細分析はスキップします（APIのコスト節約）。
- **点数が高い（70点以上）**：相場を動かす可能性のある重要なニュースとして、次のステップ（詳細分析）へ進みます。

### ステップ3：エージェントによる詳細分析（2次チェック）
パニック度が高かった重要なニュースについて、より高度な推論ができるAI（Geminiエージェント）が「プロのアナリスト」として詳細な分析を行います。

ここでAIエージェントは以下の「思考プロセス」を自動で行います：
1. **企業名の特定**：ニュースの主役となっている企業を見つけ出します。
2. **証券コードの検索**：その企業の証券コード（例：ソニーグループなら「6758」）をシステム内の検索ツールを使って正確に調べます。
3. **投資判断**：「このニュースで株価はどうなるか？」を考え、「買うべき（BUY）」「売るべき（SELL）」「様子見（HOLD）」の結論と、その具体的な理由を導き出します。

### ステップ4：最新の株価データの取得（事実確認）
ステップ3でAIが証券コードを特定できた場合、システムは株式市場のリアルタイムデータ（Yahoo Finance等）にアクセスし、「現在の株価」と「前日からの変動率（何%上がったか/下がったか）」を取得します。
取得できたリアルタイムな株価情報を、AIの判断理由の先頭に付け加えます。これにより、「実際に株価が下がっている中でのBUY判断なのか」などが一目でわかるようになります。

### ステップ5：記録と通知（レポート作成）
ここまでの全自動で行われた分析結果をまとめて記録・報告します。
- **スプレッドシートへ記録**：分析したすべてのニュースの結果（パニック度、AIの投資判断、理由、株価情報など）をGoogleスプレッドシートにリストとして書き込みます。
- **重要アラートのメール送信**：AIが「買うべき（BUY）」と判断し、かつパニック度が非常に高い（85点以上）という「超重要ニュース」だった場合のみ、あらかじめ設定した担当者のGmail宛てに速報メール（アラート）を自動送信します。

---

### まとめ
以前の「ニュース取得 → 分析実行 → スプレッドシート書き込み」というシンプルな流れから進化し、現在は以下のようになっています。

1. **ニュース取得**
2. **AIによる1次チェック**（ふるい落とし）
3. **AIエージェントによる深掘り分析**（証券コードの検索ツールなどもAI自身が使用）
4. **リアルタイム株価データとの連携**
5. **スプレッドシート書き込み ＆ アラート通知**

これにより、単にニュースを分析するだけでなく、より現実の市場に即した実用的なAIアシスタントとして機能します。

### ステップ6：GitHub Actionsによる自動実行（スケジュール＆休日スキップ）
以前はローカル（自分のPC）で常に起動しておく必要がありましたが、現在は**GitHub Actions**というクラウドの仕組みを使って、決まった時間に自動で動くようになっています。
- **平日のみ実行**：土日や日本の祝日など、株式市場が動いていない日は自動的に判断して処理をスキップします。無駄なAPI呼び出しを防ぎます。
- **完全自動化**：PCを閉じていても、クラウド上で指定したスケジュールに従って上記のステップ1〜5が実行されます。
